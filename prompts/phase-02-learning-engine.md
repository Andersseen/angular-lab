# Phase 02 — Learning Engine

## Goal

Build the core learning engine that renders missions, tracks progress, and presents content types.

## Deliverables

- Mission catalog page.
- Mission detail page with step navigation.
- Mission state management.
- Content type renderers for concept, comparison, exercise, example, and checkpoint.
- Local browser storage for progress.
- Reset mission behavior.

## Constraints

- Follow the specs in `specs/` exactly.
- Do not add real lesson content yet; use placeholder missions.
- Keep components small and testable.
- Verify behavior with component and E2E tests.
- Do not add authentication, payments, backend logic, or gamification.

## Notes

- The code editor used in exercises will be added in Phase 03, so exercises in this phase may use a simple text area placeholder.
- Focus on the data model and navigation flow, not visual polish.

# Learning Model Specification — Angular Lab

## Learning Philosophy

Learning happens best when learners alternate between short explanations, concrete examples, and immediate practice. Angular Lab structures every mission around this cycle.

## Progression Model

### Tracks

- A track is a broad learning area, such as "Fundamentals," "Reactivity," or "Routing."
- Tracks are ordered from beginner to advanced.
- A learner may follow the suggested order or jump between tracks.

### Missions

- A mission is a self-contained learning unit.
- Each mission has a clear goal, such as "Build a searchable list" or "Handle form validation."
- Missions are composed of a sequence of steps.
- A mission is complete when the learner reaches the final checkpoint.

### Steps

- A step is the smallest unit of learning.
- A step presents one idea or one task.
- Steps must be completable in a few minutes.
- Each step ends with a clear transition to the next step.

## Engagement Patterns

### Explain

- Present one concept at a time.
- Use plain language before introducing terminology.
- Anchor new ideas to code the learner has already seen.

### Show

- Provide a minimal, runnable example.
- Highlight the part of the code that matters.
- Allow the learner to modify values and see the effect.

### Practice

- Ask the learner to write or change code.
- Give immediate feedback based on the output.
- Offer hints after the first failed attempt.

### Compare

- Show two valid approaches side by side.
- Explain when to prefer each.
- Do not present one approach as always correct.

## Feedback Rules

- Feedback must focus on the learner's visible output, not on hidden implementation details.
- Error messages must explain what happened and suggest a next step.
- Correct submissions must confirm what the learner did right.
- Hints must become more specific with each failed attempt.

## Spacing and Review

- Previously learned concepts reappear in later missions as review steps.
- Review steps use slightly different contexts to reinforce transfer.
- Learners can revisit any completed mission.

## Accessibility

- Instructions must be readable by assistive technology.
- Interactive exercises must provide clear labels and focus management.
- Motion in examples must respect reduced-motion preferences.
